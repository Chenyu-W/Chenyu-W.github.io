【笔记】杂题乱做

发表于 2023-05-09 更新于 2023-07-08 分类于 笔记

You and me, chasing down forever after.

CF1256E Beautiful Mirrors

期望 DP，倒着不行就正过来，设 \(f_i\) 为从 \(1\) 走到 \(i\) 且 \(i\) 通过了的期望，显然地，有 \[ f_i = \dfrac{p_i}{100}(f_{i - 1} + 1) + \dfrac{100 - p_i}{100}(f_{i - 1} + 1 + f_i)\\ f_i = \dfrac{100}{p_i}(f_{i - 1} + 1) \] \(O(n)\) 递推即可。Code

CF14D Two Paths

枚举断开边，求两个连通块的直径之积即可。

时间复杂度 \(O(n^2)\)。Code

CF459E Pashmak and Graph

考虑按边权进行排序，若非严格递增，则有 \(f_v = \max(f_v, f_u + 1)\)，严格递增只需将边权一样的放一起 dp 即可。

时间复杂度 \(O(n\log n)\)。Code

CF900D Unusual Sequences

首先显然 \(x | y\) 才有解，令 \(n = \dfrac{y}{x}\)，那么问题就变为了使 \(\gcd(a_1, \ldots, a_x) = 1\)，\(\sum_ia_i = n\)。

先不考虑 \(\gcd = 1\) 的限制，用插板法方案为 \(f(n) = \sum_x\binom{n - 1}{x - 1} = 2^{n - 1}\)，然后容斥得到答案为 \(\sum_{d | n}\mu(d)f(d)\)，于是对 \(n\) 的所有质因子做容斥即可。

时间复杂度 \(O(\sqrt n)\)。Code

CF797E Array Queries

这玩意长得就像根号分治。考虑对于 \(k \le \sqrt{n}\)，我们可以 dp 出所有 \(p, k\) 的答案，这是 \(O(n\sqrt{n})\) 的。

对于 \(k > \sqrt{n}\)，答案显然不超过 \(\sqrt{n}\)，直接暴力即可。

时间复杂度 \(O(n\sqrt{n})\)。Code

CF109C Lucky Tree

考虑把关键边断掉，然后对于每一个大小为 \(m\) 的连通块，其贡献为 \(m(n - m)(n - m - 1)\)，把所有贡献加起来即可。

时间复杂度 \(O(n)\)。Code

Luogu3760 异或和

首先求出前缀和，然后就是转化为求 \(\oplus_{0 < l < r \le n} s_r - s_l\)，于是我们可以拆位，这样就变为对于每一个二进制位 \(k\)，求有多少个 \(s_r - s_l\) 的第 \(k\) 位为 \(1\)。

我们考虑枚举右端点，假设当前枚举到 \(s_i\)，我们分两种情况讨论。

如果 \(s_i\) 的第 \(k\) 位为 \(1\)，那么能造成贡献的 \(s_j\) 肯定要满足 \(s_j\) 的第 \(k\) 位为 \(1\)，\(s_j\) 第 \(k\) 位后面的数小于 \(s_i\) 第 \(k\) 位后面的数；或 \(s_j\) 的第 \(k\) 位为 \(0\)，\(s_j\) 第 \(k\) 位后面的数大于 \(s_i\) 第 \(k\) 位后面的数。\(s_i\) 的第 \(k\) 位同理，于是我们开两个树状数组维护即可。

时间复杂度 \(O(n\log \sum a\log n)\)。Code

20230514模拟赛T1 最大公约数

给定 \(n, m, l, r\)，求有多少 \(a_1, a_2, \ldots, a_n\)，满足 \(a_i \in [1, m]\)，且 \(\gcd(a_1, a_2,\ldots, a_n) \in [l, r]\)。\(1 \le n, m, l, r \le 10^6\)

求单次 \(O(\sqrt{m})\) 做法。

因为 \(r - l\) 不大，所以我们考虑枚举 \(\gcd\)，假设我们现在要算 \(\gcd = x\) 的方案数。那就相当于 \(a_i \in \left[1, \left\lfloor\dfrac{m}{x}\right\rfloor\right]\)，且 \(\gcd = 1\)，我们先不考虑 \(\gcd = 1\) 的限制，得到此时答案为 \(\left\lfloor\dfrac{m}{x}\right\rfloor^n\)，然后依旧是利用莫比乌斯函数容斥一下即可，最终我们可以得到答案为 \[ \sum_{i = l}^r\sum_{d = 1}^{\lfloor\frac{m}{i}\rfloor} \mu(d)\left\lfloor\dfrac{m}{id}\right\rfloor^n \] 直接对着式子写即可，时间复杂度 \(O(m\log m)\)。Code

CF1082E Increasing Frequency

简单题，设 \(pre_i\) 表示上一个和 \(a_i\) 相同的位置，\(f_i\) 表示 \([1, i]\) 中有多少和 \(a_i\) 相同的数，\(g_i\) 表示 \([1, i]\) 中有多少个 \(c\)。然后我们考虑枚举 \(i\)，统计操作右端点为 \(i\) 时的答案，显然地，我们会贪心的把左端点选在和 \(a_i\) 相同的位置上，我们设这个位置是 \(j\)，那么此时的答案即为 \(g_n + (f_i - f_{pre_j}) - (g_i - g_{j - 1})\)（这里是通过计算增加的贡献来统计答案），然后我们发现其实只要求 \(f_{pre_j} - g_{j - 1}\) 的最小值即可，我们对于每一个 \(a_i\) 都开桶维护最小值即可。

时间复杂度 \(O(n)\)。Code

CF859C Square Subsets

观察到 \([1, 70]\) 只有 \(18\) 个质数，于是考虑状压，用一个状态 \(b_i\) 来表示 \(a_i\)，\(b_i\) 的第 \(j\) 位为 \(1\) 当且仅当 \(a_i\) 被分解后第 \(j\) 个质数的指数为奇数。这样子集积为平方数就等价于在 \(b\) 中求异或和为 \(0\) 的子集，直接线性基即可。

时间复杂度 \(O(\pi(a)n)\)。

CF662A Gambling Nim

一个挺巧妙的转化。

假设我们这次随机到了 \(a_{p_1}, \ldots, a_{p_m}\) 和 \(b_{p_{m + 1}}, \ldots, b_{p_n}\)，那么根据 Nim 游戏的结论，先手必败当且仅当 \[ a_{p_1} \oplus \cdots \oplus a_{p_m} \oplus b_{p_{m + 1}} \oplus \cdots b_{p_n} = 0 \] 这个式子变化的量太多了，不是很好处理，所以我们考虑左右两边同时异或上 \(\oplus_{i = 1}^n a_i\)，这样式子就变为 \[ (b_{p_{m + 1}} \oplus a_{p_{m + 1}}) \oplus \cdots \oplus (b_{p_n} \oplus a_{p_n}) = \oplus_{i = 1}^n a_i \] 我们记 \(A = \oplus_{i = 1}^n a_i\)，\(S = \{a_i \oplus b_i\}\)，这样我们的问题就变为了从 \(S\) 中选一个子集异或和为 \(A\) 的概率。

显然把线性基搞出来，记线性基大小为 \(sz\)，那么如果 \(A\) 不能被表示出来，概率显然为 \(1\)，如果 \(A\) 能被表示出来，概率为 \(1 - \dfrac{2^{n - sz}}{2^n} = \dfrac{2^{sz} - 1}{2^{sz}}\)。

时间复杂度 \(O(n\log V)\)。Code

CF883I Photo Processing

首先二分答案转为判定。

我们记当前二分的值为 \(x\)，考虑一个 dp 的判定，首先将 \(a\) 进行排序，然后记 \(f_i\) 表示以 \(i\) 结尾是否合法，那么我们显然有转移 \(f_i = [\sum_{j \in [0, i - m], a_{j + 1} \ge a_i - x} f_j > 0]\)。这样直接转移是 \(O(n^2)\) 的。观察到对于每一个 \(i\)，转移的 \(j\) 是一个连续的区间，所以可以使用树状数组优化至 \(O(n\log^2n)\)。Code

20230524模拟赛T2 表达式求值

给定 \(a_1, \ldots, a_n\)，对于 \(\forall i \in [1, n)\)，在 \(a_i\) 和 \(a_{i + 1}\) 放一个运算符号 \(opt_i \in \{+, -, \times\}\)，你可以任意指定运算顺序，求这个表达式的值，两次运算不同当且仅当某一次选取的两个数不同，求所有情况得到的结果之和。\(1 \le n \le 500\)

显然的区间 dp。设 \(f_{l, r}\) 表示区间 \([l, r]\) 所有情况下的结果之和，不难得到转移 \[ f_{l, r} = \sum_{k = l}^{r - 1} \begin{cases} \binom{r - l - 1}{k - l}[f_{l, k}(r - k - 1)! + f_{k + 1, r}(k - l)!] & opt_k = + \\ \binom{r - l - 1}{k - l}[f_{l, k}(r - k - 1)! - f_{k + 1, r}(k - l)!] & opt_k = - \\ \binom{r - l - 1}{k - l}f_{l, k}f_{k + 1, r} & opt_k = \times \end{cases} \] 稍微解释一下，我们计算 \(f_{l, r}\) 的时候，考虑枚举中间所有的符号 \(k\)，然后钦定 \(opt_k\) 是最后一个计算的，然后我们考虑对于每一个 \(k\) 如何计算答案。

对于 \(opt_k = \times\)，左右的贡献显然是 \(f_{l, k}f_{k +1, r}\)。

对于 \(opt_k = +\)，我们直接 \(f_{l, k} + f_{k + 1, r}\) 显然是错的，左右和的个数都不一样肯定不行，左边的每个和都要和右边的和做一次加法，而右边每个一共有 \((r - k - 1)!\) 个和，同理，右边的每个和重复出现了 \((k - l)!\) 次，所以答案为 \(f_{l, k}(r - k - 1)! + f_{k + 1, r}(k - l)!\)。

对于 \(opt_k = -\) 本质和 \(opt_k = +\) 一致。

但是这样我们会漏考虑一些东西，因为计算的顺序不同会重复贡献多次答案，而我们只统计了一次，所以我们每次再乘上 \(\binom{r - l - 1}{k - l}\)。

时间复杂度 \(O(n^3)\)。

HDU6125 Free from square

称 \(\le 19\) 的质数为小质数，其余为大质数。

没有平方因子本质上就是每个素因子最多出现一次，所以我们考虑状压，设 \(st_i\) 表示第 \(i\) 个数所代表的状态，\(f_{i, S}\) 表示选了 \(i\) 个，状态集合为 \(S\) 的方案数，容易得到转移 \[ f_{i + 1, S | st_j} = f_{i + 1, s| st_j} + f_{j, S}[S \& st_j = 0] \] 但这样显然是会寄的，观察到每个数分解完最多只会有一个大质数，所以我们可以对小质数状压，对大指数进行分组背包，这样就可以了。

时间复杂度 \(O(2^8nk)\)。Code

CF594D REQ

首先考虑欧拉函数的计算式 \(\varphi(n) = n\prod_{p | n} \dfrac{p - 1}{p}\)，所以我们考虑对每一个 \(a_i\) 进行因式分解，然后计算每个质因子的贡献。

不难发现我们考虑一下重复质因子的贡献就做完了，这玩意在线不太好做，所以我们考虑离线。将询问按右端点排序，然后考虑一个一个加入，假设我们现在要在第 \(i\) 个位置加入一个质因子 \(p\)，我们用 \(last_p\) 表示 \(p\) 上次出现的位置，然后我们在 \(last_p\) 上乘一个 \(\dfrac{p}{p - 1}\)，再在 \(i\) 上乘一个 \(\dfrac{p - 1}{p}\) 即可，注意更新 \(last_p\) 的值。

开个树状数组维护区间乘积就做完了，时间复杂度 \(O(n\log n\log V)\)。Code

CF724G Xor-matic Number of the Graph

写复杂了/kk。

首先需要知道一个事情，就是两个点间所有路径的异或都可以表示为这两点的一条简单路径异或若干个环。

所以考虑将环的的异或和加到一个线性基里，记 \(s_i\) 表示 \(1 \rightarrow i\) 一条简单路径的异或和。根据上述结论，我们发现两个点 \(i, j\) 之间的路径异或和可以表示为 \(s_i \oplus s_j \oplus x\) 其中 \(x\) 是线性基里的一个子集，这样我们枚举点可以做到 \(O(n^2\log V)\)。

枚举点是不优的，所以考虑计算每一个二进制位的贡献，记当前二进制为 \(w\)，\(s\) 中第 \(w\) 位为 \(1\) 的有 \(o\) 个，第 \(w\) 位为 \(0\) 的有 \(z\) 个。线性基中第 \(w\) 位为 \(1\) 的有 \(op\) 个，第 \(w\) 位为 \(0\) 的有 \(zp\) 个。

假设 \(s_i \oplus s_j\) 的第 \(w\) 位为 \(0\)，那么这样的 \((i, j)\) 共有 \(\binom{o}{2} + \binom{z}{2}\) 种，如果产生贡献，则 \(op\) 中要选奇数个，这样有 \(2^{op - 1}\) 种选法，所以这样一共就有 \(2^{zp}2^{op - 1}(\binom{o}{2}+ \binom{z}{2})\) 种情况。

假设 \(s_i \oplus s_j\) 的第 \(w\) 为为 \(1\)，那么这样的 \((i, j)\) 共有 \(oz\) 种，如果产生贡献，则 \(op\) 中要选偶数个，也有 \(2^{op - 1}\) 种选法，这样一共有 \(2^{zp}2^{op - 1}oz\) 中情况。

注意当 \(op = 0\) 时，第一种情况不存在，第二种情况的方法数为 \(2^{zp}oz\)。

这样直接对着式子写即可，时间复杂度 \(O(n\log^2V)\)。Code

20230528模拟赛T3 直径

给一棵 \(n\) 个点的正权边树，\(q\) 次询问。每次询问断掉树中的 \(k\) 条边，求所有连通块直径之和。\(1 \le n, q, \sum k \le 10^5\)。

简单但难写的 DS 题，场上写了 1.5h。

首先正权边的树有一个性质，就是两个连通块合并在一起的直径的两个端点一定取自于两个连通块直径的那四个端点，利用这个性质，我们可以很容易用数据结构对若干个区间的直径进行合并。

考虑从树上删边的过程相当于每次丢掉一个子树，对应到 dfs 上就是一个区间，而剩下的连通块就对应剩下区间的并，所以我们从根开始，将每个删除的子树先递归处理，然后把剩下的区间用上述方法合并起来，这样总共合并 \(O(\sum k)\) 次，所以时间复杂度为 \(O(\sum k\log n)\)。Code

CF1483C Skyline Photo

设 \(f_i\) 表示划分前 \(i\) 个数的最大值，那么显然有转移 \(f_i = \max_{j \in [1, i - 1]} f_{j - 1} + val(j, i)\) 其中 \(val\) 表示一段区间的价值。这样直接做是 \(O(n^2)\) 的。

我们记 \(pos_i\) 表示 \(i\) 左边第一个小于 \(i\) 的下标，不难发现如果我们取 \(j \le pos_i\) 进行转移，那么得到的结果都是 \(f_{pos_i}\)，所以我们可以把转移改为 \(f_i = \max\{f_{pos_i}, \max_{j \in [pos_i, i - 1]} f_j + b_i\}\)，后半部分简单线段树维护一下即可。

时间复杂度 \(O(n\log n)\)。Code

CF1023F Mobile Phone Network

典题。因为 \(k\) 条边都要在最终的生成树里面，所以我们不妨直接把它们放进一棵树里，然后对剩下的边用 kruscal 的方式补全，不难发现在这棵树里面的边不会对答案造成影响，而对于剩下的边 \((u, v, w)\)，生成树上 \(u \rightarrow v\) 路径上的边的边权一定要小于等于 \(w\)，于是树剖区间覆盖即可。

时间复杂度 \(O(n\log^2 n)\)。Code

ABC163F path pass i

很像 CF1681F。我们考虑对于一个颜色 \(k\) 怎么计算答案，我们把带有 \(k\) 的边全部断掉，然后剩下每个连通块的点数为 \(sz_i\)，我们用容斥就可以得到最终的答案为 \(\frac{n(n - 1)}{2} - \sum\frac{sz_i(sz_i - 1)}{2}\)，再算上形如 \((u, u)\) 的路径条数即可，对于这种断边的问题，我们可以用线段树分治解决。时间复杂度 \(O(n\log^2n)\)。Code

ABC249E RLE

设 \(f_{i, j}\) 为前 \(i\) 位压缩后长度为 \(j\) 的方案数，枚举最后一段极长连续段 \([k, i]\) 可得转移方程 \[ f_{i, j} = \sum_{k = 1}^i 25 \cdot f_{k - 1, j - \lfloor{\log_{10}(i - k + 1)}\rfloor - 2} \] 初始化为 \(f_{i, \lfloor\log_{10} i\rfloor + 2} = 26\)。这样直接写是 \(O(n ^ 3)\) 的。但是我们发现 \(\lfloor{\log_{10}(i - k + 1)}\rfloor\) 的值域很小，所以我们考虑枚举这个东西，这样转移方程就变为 \[ f_{i, j} = 25\sum_{k = 0}^{\lfloor\log_{10} i\rfloor}\sum_{l = i - 10^{k + 1} + 1}^{i - 10^k} f_{l, j - k - 2} \] 第二个和式用前缀和维护即可，时间复杂度 \(O(n^2\log_{10}{n})\)。Code

CF543D Road Improvement

记 \(f_i\) 为 \(i\) 到子树内点都满足要求的方案数，\(g_i\) 为 \(i\) 到子树外点都满足要求的方案数，那么对于最后的每个点的答案就是 \(f_i \cdot g_i\)。

首先考虑 \(f_i\)，我们显然有转移 \(f_u = \prod_{v \in \text{son}_u} (f_v + 1)\)。然后我们考虑换根计算 \(g_i\)，我们考虑当前计算的是 \(g_u\)，那么我们的答案就分为两个部分，第一个部分是 \(g_{fa_u}\)，代表先从 \(u\) 走到 \(fa_u\) 然后满足条件的方案数，第二种是走到 \(fa_u\) 其他子树的点，这种情况的答案为 \(\prod_{v \in \text{son}_{fa_u}, v \neq u} (f_v + 1)\)，用一个前缀积和一个后缀积即可计算，这两个部分乘起来即可，但是这样我们没考虑所有边都修的情况，所以答案要加 \(1\)。时间复杂度 \(O(n)\)。Code

CF547C Mike and Foam

我们考虑计算每次答案的增减量，设当前在桌上牌的集合为 \(S\)，我们要加入/拿走的数为 \(x\)，我们要求的就是 \[ \begin{aligned} &\sum_{k \in S} [\gcd(x, k) = 1] \\ &= \sum_{k \in S} \sum_{d | \gcd(x, k)} \mu(d) \\ &= \sum_{d | x}\mu(d)\sum_{k \in S,d|k}1 \end{aligned} \] 开桶维护某个数倍数的出现次数即可。时间复杂度 \(O(n\log a)\)。Code

Luogu7009 Magical GCD

对于一个值域为 \(V\) 的序列，他后缀 \(\gcd\) 中不同的值只有 \(O(\log V)\) 个，因为每次加入一个数，\(\gcd\) 要么不变，要么至少减半。那么就很简单了。我们设现在考虑右端点为 \(x\) 时的答案。那么我们维护序列 \(a_1, \ldots a_x\) 后缀 \(\gcd\) 每一个连续段的左端点，我们开一个链表，每一个节点存一个二元组 \((i, j)\)，分别表示这个位置的下标和目前为止的后缀 \(\gcd\)。我们首先遍历链表，使 \(j \gets \gcd(j, a_x)\)，然后把重复元素删掉，最后加入二元组 \((x, a_x)\)，并更新答案。时间复杂度 \(O(n\log V)\)。Code

AGC038C LCMs

只需算 \(\sum_{i = 1}^n\sum_{j = 1}^n \text{lcm}(a_i, a_j)\)。

记 \(c_i\) 为 \(i\) 的出现次数，\(m\) 为所有数中最大值，那么就是求 \(\sum_{i = 1}^m\sum_{j = 1}^m \text{lcm}(i, j) c_ic_j\)。

运用莫反基本技术稍微推推得到 \[ \sum_{d = 1}^m\mu(d)d^2\sum_{k = 1}^{\lfloor\frac{m}{d}\rfloor}k\left(\sum_{i=1}^{\lfloor\frac{m}{kd}\rfloor}i\cdot c_{idk}\right)^2 \] 令 \(T = kd\)。稍微推推得到 \[ \sum_{T = 1}^m T\sum_{d|T}\mu(d)\cdot d\left(\sum_{i = 1}^{\lfloor\frac{m}{T}\rfloor}i \cdot c_{iT}\right)^2 \] 这样就可以 \(O(m\log m)\) 算啦。Code
