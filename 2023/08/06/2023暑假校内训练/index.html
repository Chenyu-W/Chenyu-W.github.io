0371T4

给定一个长度为 \(n\) 的序列，\(q\) 次询问，每次询问给定 \([l, r]\)，求一个最长的子区间 \([l', r']\)，使得 \([l', r']\) 中出现数的种类小于 \([l, r]\) 中出现数的种类。\(1 \le n, q \le 10^5\)。

场上情况：想出来了，但是来不及写了。

考虑一个数 \(a_i\) 怎么样不会出现在一个子区间里。

  1. \(a_i\) 在区间里第一次出现，此时取最长子区间 \([l, i - 1]\)。
  2. \(a_i\) 在区间内最后一次出现，此时取最长子区间 \([i + 1, r]\)。
  3. \(a_i\) 不是上述两种情况，此时取最长子区间 \([pre_i + 1, i - 1]\)。

分三类做二维偏序即可。时间复杂度 \(O(n\log n)\)。

0801T2

给定一棵 \(n\) 个点的树，一些节点上有人，有 \(k\) 秒活动时间，每秒一个人可以待在原地或者跳到父节点，最小化有人的节点个数。\(1 \le n, k \le 10^5\)

场上情况：切了。

考虑贪心，假设我们现在已经处理完了 \(u\) 的子树，现在要处理 \(u\) 的儿子往 \(u\) 上跳的情况，那么显然对于一个儿子 \(v\)，现在处于 \(v\) 上的所有人都能跳到 \(u\) 才会优。所以我们记录当前处于 \(v\) 上的最大深度，看是否往上跳即可。

0802T3

给定一棵 \(n\) 个结点的树，一个人初始在 \(1\) 号结点，每秒随机走向和当前点有边的结点，求走到 \(n\) 号结点的期望时间。\(1 \le n \le 10^5\)

场上情况：切了。

好像有更强支持多测的做法，但是我场上只想到了下面的东西。

考虑以 \(n\) 为根。设 \(f_i\) 表示从 \(i\) 走到 \(n\) 的期望时间，显然地，我们有 \[ f_u = \dfrac{1}{d_u}\left(\sum_{v \in \text{son}_u} f_v + f_{fa_u}\right) + 1 \] 高斯消元可以做到 \(O(n^3)\)。

我们考虑从叶子结点往上推的过程，我们发现每个 \(f_u\) 都可以表示为 \(kf_{fa_u} + b\) 的形式，类似 dp 地取维护 \(k, b\) 即可。时间复杂度 \(O(n)\)。Code

0804T2

原题

场上情况：切了。

只有前三种操作是编辑距离板子，考虑操作四。

注意题目给了一个奇怪的限制，仔细想想，这保证了每个字符最多只能被交换一次。同时一个显然的事情是，我们只会交换原有的字符，否则新加进来直接放在正确的位置上显然更优。

于是设 \(f_{i, j}\) 表示 \(A\) 的前 \(i\) 位和 \(B\) 前 \(j\) 位匹配的最小代价。我们考虑将 \(i\) 位置和 \(k\) 位置进行交换，然后再在他们中间插入若干个字符，我们设最后 \(A_i, \ldots, A_k\) 这一段要和 \(B_l, \ldots, B_j\) 这一段匹配，所以我们需要 \(A_i = B_l\)，\(A_k = B_j\)。此时我们可以得到转移 \[ f_{i, j} = \min_{A_i = B_l, A_k = B_j} f_{k - 1, l - 1} + (i - k - 1)b + (j - l - 1)a + d \] 这样似乎是 \(O(n^4)\) 的，但是我们不难发现每次转移只会取最大的 \(k\) 和 \(l\)，简单证明一下。

设 \(k_1, l_1\) 是另外的一组转移，那么我们有 \[ \begin{aligned} &f_{k - 1, l - 1} + (i - k - 1)b + (j - l - 1)a + d - [f_{k_1 - 1, l_1 - 1} + (i - k_1 - 1)b + (j - l_1 - 1)a + d] \\ &= f_{k - 1, l - 1} - f_{k_1 - 1, l_1 - 1} - (k - k_1)a - (l - l_1)b \end{aligned} \] 我们考虑 \(f_{k - 1, l - 1} - f_{k_1 - 1, l_1 - 1}\) 的意义，就是已经拼好了前面，要拼到后面还要花费的最小的代价，不难发现 \((k - k_1)b + (l - l_1)a\) 是一种可行的代价，根据 \(f\) 的最小性，我们可以判断前者小于后者，所以原式的值 \(\le 0\)。

所以我么可以预处理处每个位置的转移点，然后就可以做到 \(O(n^2)\) 了。

0805T4

给定长度为 \(n\) 字符串 \(S\)，记 \(f_i = \sum_{j = 1}^{i - 1}[S[1:j] = S[i - j + 1: i]]\gcd(i,j)\)，求 \(\prod_{i = 1}^n(f_i + 1) \bmod 998244353\)，\(1 \le n \le 10^6\)

场上情况：首 A。

考虑建失配树，满足条件的 \(j\) 为 \(i\) 到根节点路径上的点。

看见 \(\gcd\)，想到欧拉反演，于是 \[ \begin{aligned} &f_i = \sum_{j}\gcd(i, j)\\ &=\sum_{j}\sum_{d|\gcd(i, j)}\varphi(d)\\ &=\sum_{d|i}\varphi(d)\sum_{d|j} 1 \end{aligned} \] 第二个和式开桶记录因数出现次数即可，时间复杂度 \(O(n\log n)\)。

0805T2

给定长度为 \(n\) 的序列 \(A\)，记 \(B_{l, r}\) 为 \(A\) 区间 \([l, r]\) 中的数排完序的序列，\(m\) 次询问，求 \(B_{l, r}\) 下标为奇数的数的和。\(1 \le n, m \le 10^5\)。时限 16s。

场上情况：没取模 \(100 \to 0\)。

巨大多时限，考虑 \(O(n\sqrt n\log n)\) 算法。考虑莫队，每次新加入/删除一个数相当于在值域上插入/删除一个值，权值线段树维护下标为奇数的数之和即可。时间复杂度 \(O(n\sqrt n\log n)\)。

0807T3

给定一棵 \(n\) 个结点的树，第 \(i\) 条边有 \(p_i\) 的概率断掉，求根节点所在连通块大小的期望和方差。\(1 \le n \le 1.5 \times 10^6\)。

场上情况：写了期望 \(40\) 跑路。

设 \(x_i\) 表示 \(i\) 号结点是否对答案产生贡献，不难发现 \(E(x_i)\) 即为 \(i\) 到根节点上所有边的概率的乘积，于是期望就做完了。

方差的做法是根据 \(D(X) = E(X^2) - E^2(X)\)（场上不知道这玩意，wtcl）。其中 \(E^2(X)\) 上一问求完了，于是我们考虑 \[ E(X^2) = E\left(\sum_{i, j} x_ix_j\right) = \sum_{i, j}\dfrac{E(x_i)E(x_j)}{E(x_{\text{LCA(i, j)}})} \] 在 \(\text{LCA}\) 处统计答案即可。时间复杂度 \(O(n)\)。

0808T0

有两个可重集 \(S_1, S_2\)，集合内的元素为二元组，\(m\) 次操作，支持插入，删除，查询

\[ \min_{(A_1, B_1) \in S_1, (A_2, B_2) \in S_2} \max\{A_1 + A_2, B_1 + B_2\} \]

场上情况：线段树分治两个栈用一个栈顶一起退栈，\(100 \to 0\)。

考虑没有删除怎么做，我们设现在新加入一个二元组 \((x, y)\)，那么要求的是 \[ \min_{(a, b) \in S} \max\{a + x, b + y\} \] 把 \(\max\) 拆开，变成了两个二维偏序，线段树维护即可。

删除再套一个线段树分治，时间复杂度 \(O(n\log^2n)\)。

0808T4

有一条长度为 \(n + 1\) 的数轴，数轴上有 \(m\) 个点，坐标为 \(a_1, \ldots, a_m\)，每秒随机挑选一个点往前走一步，两个点重复就会合并，一个点走到 \(n + 1\) 会消失，求所有点全都消失的时间的期望。\(1 \le m, n \le 1000\)。

场上情况：没想到。

记 \(a_{m + 1} = n + 1\)，不难发现所有点全部消失的过程相当于每个点不断和下一个点合并的过程，于是记 \(f_{i, j}\) 表示坐标为 \(i\) 的点和坐标为 \(j\) 的点合并 \(i\) 走的步数的期望，最终答案即为 \(\sum_{i = 1}^m f_{a_i, a_{i + 1}}\)。\(f_{i, j}\) 的转移是容易的。时间复杂度 \(O(n^2)\)。

0809T3

原题

场上情况：切了。

首先考虑不加边，就是换根 dp 板子。然后我们考虑加边会给答案造成多少贡献，设当前根为 \(r\)，我们从 \(r\) 两棵不同的子树中选出两个点连边，那么我们观察到每个点的贡献可以摊到他祖先上每个点都进行计算，所以答案为 \(\sum_{i 在环上, i \neq r} sz_i\)。所以其实就是对于每一个点 \(u\)，选两条从上往下的链，使得上面那个东西最小，记最小值和次小值即可。容易换根 dp 转移，时间复杂度 \(O(n)\)。

0810T3

给定一个由字符 C 和 T 组成的字符串 \(S\)。\(q\) 次询问，每次询问给定 \([l, r]\)，求最少删除几个字符，才能使 \(S[l : r]\) 中每一个前缀和每一个后缀，其 C 的个数不小于 T 的个数。\(1 \le n, q \le 5 \times 10^5\)。

将 C 看作 \(1\)，T看作 \(-1\)，限制等价于任意前缀和与任意后缀和非负，记 \(s_i\) 为序列的前缀和。

有一个显然正确的贪心，先满足前缀限制，代价为前缀最小值的绝对值，然后再在此基础上算一遍后缀。

考虑如何优化，记 \(f(x) = -\min_{i \in [l, x]}\{s_x - s_{l - 1}\}\) 为将 \([l, x]\) 满足前缀限制的最小代价，那么满足前缀限制的代价就是 \(f(r)\)，然后在考虑计算后缀最小值，考虑在 \(x\) 位置的后缀和，为 \(g(x) = s_r - s_{x - 1} + f(r) - f(x - 1)\)，意思是之前的值加上删去的数。然后答案即为 \(f(r) - \min_{x \in [l, r]} g(x)\)，将柿子展开，发现答案是 \[ \max_{l - 1 \le x \le y \le r} \{s_y - s_x\} - s_r + s_{l - 1} \] 线段树维护最大子段和即可，时间复杂度 \(O(n\log n)\)。

0810T2

原题

感谢 LJ07 不厌其烦地教了我三遍，终于懂了。

场上情况：摆摆摆。

我们考虑对于每一个 \(b_i\) 如何贪心地去匹配，首先我们肯定会贪心地选取一个 \(d_j\) 最小的，满足 \(a_i = c_j\)，且 \(j\) 还未被匹配的去和他匹配，如果不存在，我们可以随机匹配，但是随机配显然是错的，所以我们可以维护一个二分图 \(G\)，然后把这样的失配点丢进 \(G\) 里面。所以我们每次给 \(i\) 匹配的时候就相当于从 \(G\) 中删去一个点 \(j\)。在进一步会发现，就算存在这样的 \(j\)，\(i\) 也不一定能和他匹配，因为我们不确定删去了 \(j\) 之后 \(G\) 是否还存在完美匹配，根据 Hall 定理，等价于 \(\min(|N(S)| - |S|) \ge 0\)，更进一步，因为 \(G\) 中每一个点都会向他所有前缀连边，所以我们其实不用取所有的 \(S\)，只需要判断前缀最小是否非负即可，每次加入/删除一个点都会影响一个前缀的 \(|N(S)| - |S|\)，所以我们使用线段树区间加，区间最小值即可。时间复杂度 \(O(n\log n)\)。

CF1842G

考虑用所有情况的和除以情况数来进行计算。

我们最后的答案肯定长这样 \(\prod_i (a_i + v + \cdots +v)\)，相当于我们从每个括号里面挑一个然后乘起来再求和。

记 \(f_{i, j}\) 表示考虑到前 \(i\) 个括号，已经使用了 \(j\) 个操作的 \(v\)，所有情况的和。

我们考虑从当前括号选取什么来造成贡献。

  1. 选择 \(a_i\)，那么 \(f_{i, j} \gets f_{i - 1, j} \cdot a_i\)。
  2. 选择一个在这 \(j\) 次操作中带来的 \(v\)，那么 \(f_{i, j} \gets f_{i - 1, j} \cdot v\)
  3. 新选择一个操作的 \(v\)，那么我们可以从剩下的 \(m - j + 1\) 个操作中选，每个操作只要作用在 \([1, i]\) 上的点就行，因此 \(f_{i, j} \gets f_{i - 1, j - 1} \cdot (m - j + 1) \cdot i \cdot v\)。

最后考虑如何统计答案，对于 \(f_{n, j}\) 来说，我们只关心那 \(j\) 个操作使用的位置，剩下的 \(m - j\) 个操作随便选都没问题，所以贡献为 \(f_{n, j} \cdot n^{m - j}\)，最后的答案即为 \(\dfrac{\sum_{j = 0}^{\min(n, m)} f_{n, j} \cdot n^{m - j}}{n^m}\)。时间复杂度 \(O(n^2)\)。

CF1228E

考虑容斥。

记 \(f(i)\) 表示钦定 \(i\) 行最小值不为 \(1\)，且所有列最小值都为 \(1\)，那么答案为 \(\sum_{i = 0}^n(-1)^i\binom{n}{i}f(i)\)。

记 \(g(i, j)\) 表示在钦定了 \(i\) 行的基础上，钦定 \(j\) 列最小值不为 \(1\) 的方案数，那么 \(f(i) = \sum_{j = 0}^n (-1)^j\binom{n}{j}g(i, j)\)。

容易得到 \(g(i, j) = (k - 1)^{n(i + j) - ij} \cdot k^{n^2 - n(i + j) + ij}\)。

直接计算，时间复杂度 \(O(n^2)\)。

0812T2

原题

场上情况：切了。

考虑 \(O(nk)\) dp，记 \(f_{i, j, 0/1}\) 表示当前考虑了前 \(i\) 位，进了 \(j\) 次，当前是否进位的方案数，容易写出转移 \[ \begin{aligned} &f_{i, j, 0} \gets 3f_{i - 1, j, 0}\\ &f_{i, j, 0} \gets f_{i - 1, j - 1, 1}\\ &f_{i, j, 1} \gets 3f_{i - 1, j, 1}\\ &f_{i, j, 1} \gets f_{i - 1, j - 1, 0} \end{aligned} \] 不难发现，当 \(i - 1\) 的进位状态和 \(i\) 的进位状态相同时会贡献 \(3\) 的系数，否则只贡献 \(1\) 的系数。

记 \(x_i\) 表示第 \(i\) 为是否进位，特别的 \(x_0 = 0\)。我们可以枚举 \(i\)，表示有 \(i\) 个 \(j\) 满足 \(x_j \neq x_{j - 1}\)，那么就有 \(n - i\) 个满足 \(x_{j} = x_{j - 1}\) 的，所以贡献系数 \(3^{n - j}\)，然后观察到 \(x\) 序列被分为 \(i + 1\) 个连续段，那么其中就有 \(\left\lceil\dfrac{i + 1}{2}\right\rceil\) 个 \(0\) 的连续段，\(\left\lfloor\dfrac{i + 1}{2}\right\rfloor\) 个 \(1\) 的连续段，我们就相当于要把 \(n - k + 1\) 个 \(0\) 和 \(k\) 个 \(1\) 分为这么多段，所以共有 \(\displaystyle\binom{n - k}{\lceil\frac{i+1}{2}\rceil - 1} \cdot \displaystyle\binom{k - 1}{\lfloor\frac{i+1}{2}\rfloor-1}\) 种选法，然后直接把所有贡献加起来即可。时间复杂度 \(O(n)\)。

CF1794E

本质是判断可重集相等，所以考虑 Hash。

观察到题目中给的限制很像换根 dp，所以考虑用这种方式转移 Hash。

我们定义 Hash 函数 \(H(x) = \sum_i c_iP^i\)，其中 \(c_i\) 是距离为 \(i\) 的点的个数，\(P\) 是我们选的，这样的 Hash 可以用换根简单转移，于是就以 \(O(n\log n)\) 的复杂度做完了，数据有强度，要双模数。

Note: 第一眼看到想到的是给每个距离赋一个随机权，但这样不好转移，写 Hash 要满足我们的需求。
